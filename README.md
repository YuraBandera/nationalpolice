# НАШЕ Київ — сайт ГУНП м. Київ (Roblox RP)

Офіційний сайт рольового проєкту **«НАШЕ Київ»** — Головного управління Національної
поліції України в місті Києві. Next.js 14 (App Router) + TypeScript + Tailwind CSS +
Framer Motion. Із адмін-панеллю, формами заявки/скарги, новинами, галереєю та
збереженням даних у файлову базу.

---

## Швидкий старт

Потрібен **Node.js 18.17+** (краще 20+).

```bash
# 1. Встановити залежності
npm install

# 2. Створити файл змінних середовища
cp .env.example .env

# 3. Відредагувати .env (пароль адміна, Discord-вебхуки) — див. нижче

# 4. Запустити в режимі розробки
npm run dev
```

Відкрий у браузері: **http://localhost:3000**
Адмін-панель: **http://localhost:3000/admin**

### Прод-збірка

```bash
npm run build
npm run start
```

## Налаштування (.env)

```env
# Пароль для входу в /admin
ADMIN_PASSWORD=твій-надійний-пароль

# Discord-вебхуки (НЕ те саме, що запрошення на сервер!)
DISCORD_WEBHOOK_APPLICATIONS=https://discord.com/api/webhooks/...
DISCORD_WEBHOOK_COMPLAINTS=https://discord.com/api/webhooks/...

# Публічне запрошення на Discord-сервер (кнопки на сайті)
NEXT_PUBLIC_DISCORD_INVITE=https://discord.gg/N9gFDMs9v
```

### Як отримати Discord-вебхук

1. У Discord відкрий канал → **Налаштування каналу → Інтеграції → Вебхуки**.
2. **Новий вебхук** → скопіюй **URL вебхука**.
3. Встав цей URL у `DISCORD_WEBHOOK_APPLICATIONS` (для заявок) і/або
   `DISCORD_WEBHOOK_COMPLAINTS` (для скарг).

> Це саме адреса вигляду `https://discord.com/api/webhooks/…`, а **не**
> посилання-запрошення `discord.gg/…`. Якщо вебхук не вказано — форми все одно
> зберігаються в базі, просто без сповіщення в Discord.

## Відео на головній сторінці

Поклади у папку `public/` два файли:

- `hero.mp4` — фонове відео (H.264, без звуку, ~1:30, зациклюється);
- `hero-poster.jpg` — статичний кадр-заставка (1920×1080).

Деталі — у `public/README.txt`. Без цих файлів сайт покаже градієнтний фон і
запуститься коректно.

## Адмін-панель `/admin`

Вхід за паролем `ADMIN_PASSWORD`. Можливості:

- **Новини** — додавання, редагування, видалення;
- **Галерея** — фото та відео (за посиланням), видалення;
- **Заявки** — перегляд анкет, зміна статусу, видалення;
- **Скарги** — перегляд, зміна статусу, видалення;
- **Керівництво** — редагування карток командного складу, порядок;
- **Статистика** — лічильники на головній;
- **Контакти** — Discord, Roblox, пошта, час роботи, соцмережі.

### Важливо для деплою

Файлова база вимагає **постійної файлової системи**. Тому підходять:

- **власний VPS / хостинг** (self-host, `npm run build && npm run start`);
- **Railway / Render** із під’єднаним **volume** на папку `data/`.

На **Vercel** файлова система ефемерна — дані між запусками не збережуться.
Для Vercel винеси стор у зовнішню БД (Postgres/Neon, Turso, Supabase тощо),
замінивши реалізацію в `src/lib/db.ts`. Логіка ізольована саме в цьому файлі,
тож міграція зачепить лише його.

## Структура проєкту

```
src/
  app/
    api/            # серверні роути (news, gallery, applications, complaints, ...)
    admin/          # сторінка адмін-панелі
    apply/          # сторінка заявки
    complaint/      # сторінка скарги
    news/           # сторінка всіх новин
    privacy/        # політика конфіденційності
    layout.tsx      # корінь: шрифти, toast, публічний хром
    page.tsx        # головна сторінка
  components/        # UI-компоненти (Hero, About, Units, форми, ...)
    admin/          # компоненти адмін-панелі
  lib/
    db.ts           # файлова база + seed
    discord.ts      # надсилання у Discord
    auth.ts         # авторизація адміна (cookie)
    site.ts         # підрозділи, пункти меню
    types.ts        # типи даних
data/               # тут зʼявиться db.json
public/             # hero.mp4, hero-poster.jpg, статика
```
